## Analysis

### Performance Issue

The Rust implementation shows unexpectedly poor performance for n >= 1000. The benchmark indicates significant regression compared to previous baselines.

### Potential Causes

1. **Benchmark data generation**: The Criterion benchmark may be including DataFrame construction in the timed section
2. **Polars overhead**: Column extraction from DataFrame for each operation
3. **Matrix operations**: May not be using optimal algorithms (e.g., Cholesky decomposition)

### Recommendations

1. Profile the `run_ols` function to identify bottlenecks
2. Ensure data generation is outside the benchmarked code path
3. Consider pre-computing X'X and X'y matrices
4. Review if `faer` is being used for linear algebra operations
