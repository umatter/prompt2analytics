\section{Performance benchmarks} \label{sec:performance}

This section presents detailed performance benchmarks comparing
\pkg{prompt2analytics} against established \proglang{R} packages across
41 method configurations spanning regression, panel econometrics, discrete
choice, causal inference, survival analysis, time series, statistics,
machine learning, and data munging.
Benchmark methodology follows
Section~\ref{sec:comparison}; see Appendix~\ref{app:benchmark-methodology}
for complete details.%
\footnote{A notable omission is \pkg{fixest}~\citep{Berge:2018}, which is
substantially faster than both \pkg{plm} and \pkg{lfe} for panel methods.
Informal benchmarks suggest \pkg{fixest} also outperforms \pkg{prompt2analytics}
for HDFE problems at large scale due to its highly optimized C++ implementation
of the alternating projections algorithm. Users prioritizing raw speed for
panel econometrics over natural language interfaces should consider \pkg{fixest}.}

\subsection{Benchmark results} \label{sec:benchmark-results}

We present benchmark results across nine method categories: regression,
panel econometrics, discrete choice, causal inference, survival analysis,
time series, statistics, machine learning, and data munging.
Each method is benchmarked at $n = 100{,}000$ observations with ten
iterations (median timing reported). Both \proglang{R} and \proglang{Rust}
timings measure computation only, excluding data loading and process
startup, ensuring a fair comparison of algorithmic performance.

Table~\ref{tab:benchmark-full} presents benchmark results at $n = 100{,}000$
observations.
\pkg{prompt2analytics} demonstrates substantial speedups for compute-intensive
methods: Granger causality (660$\times$), quantile regression (44$\times$),
Wilcoxon rank-sum (23$\times$), OLS with robust standard errors
(15--22$\times$ for HC0--HC3), Panel Fixed Effects (21$\times$),
group-by aggregation (21$\times$), DBSCAN (14.5$\times$),
Kruskal-Wallis (10$\times$), t-test (7$\times$),
OLS (6.9$\times$), sort (6.9$\times$), standardize (6.9$\times$),
IV/2SLS (5.2$\times$), Hausman test (4.7$\times$),
Panel Random Effects (4.5$\times$), MSTL decomposition (4.4$\times$),
Kaplan-Meier (4.1$\times$), Shapiro-Wilk (3.0$\times$),
Holt-Winters (2.6$\times$), logit (2.6$\times$), Cox proportional hazards
(2.3$\times$), and K-Means clustering (2.2$\times$).

Methods showing \proglang{R} outperforming \proglang{Rust} warrant detailed
explanation, as these cases illuminate the performance characteristics of
both approaches.

\textbf{PCA} shows the largest \proglang{R} advantage (0.03$\times$).
\proglang{R}'s \fct{prcomp} uses direct LAPACK bindings for SVD
eigendecomposition, which are highly optimized for this specific operation.
\pkg{prompt2analytics}'s implementation uses a general-purpose SVD that
incurs additional overhead. This represents a genuine algorithmic
optimization opportunity.

\textbf{Nonlinear least squares and diagnostics} show \proglang{R}
substantially faster (NLS at 0.1$\times$, diagnostics at 0.2$\times$).
\proglang{R}'s \fct{nls} benefits from decades of optimization in its
Gauss-Newton implementation with efficient C-level Jacobian computation.
Regression diagnostics involve multiple model refits that compound
implementation differences.

\textbf{ACF/PACF and chi-squared goodness of fit} show \proglang{R}
moderately faster (both 0.4$\times$). These are relatively fast
operations where \proglang{R}'s highly optimized vectorized C
implementations provide an edge.

\textbf{Near-parity methods} include AR (1.0$\times$), ANOVA two-way
(1.0$\times$), changepoint detection (0.9$\times$), HDFE (1.3$\times$),
and negative binomial regression (1.3$\times$). For these methods,
both implementations achieve comparable performance, reflecting the
dominance of core linear algebra operations where \proglang{R}'s BLAS
bindings and \proglang{Rust}'s \pkg{faer} library perform similarly.

The general pattern is that \pkg{prompt2analytics} excels at computationally
intensive iterative methods (panel fixed effects, robust standard errors,
quantile regression, DBSCAN, Wilcoxon) where \proglang{Rust}'s compile-time
optimizations and memory efficiency compound across iterations, and at
methods involving repeated model fits (Granger causality, Hausman test,
IV/2SLS). For single-pass operations dominated by optimized linear algebra,
\proglang{R}'s BLAS bindings provide comparable or superior performance.

\subsubsection{Discussion}

Performance differences reflect \proglang{Rust}'s ownership model (reduced heap
allocation), SIMD vectorization via \pkg{faer}, compiled vs. interpreted
overhead, and column-based API (no formula parsing). Speedup factors decrease
with sample size as matrix operations dominate.

These performance characteristics, combined with the evaluation results from
Section~\ref{sec:evaluation}, make fully local deployment practical. The next
section describes configurations where all components---including the language
model---run on the user's machine, eliminating external data transmission
entirely.
