\section{Comparison and validation} \label{sec:comparison}

This section compares \pkg{prompt2analytics} against established \proglang{R}
packages in terms of both performance and numerical accuracy. We demonstrate
that the \proglang{Rust} implementation achieves substantial speedups while
matching reference implementations to high precision.

%% -----------------------------------------------------------------------------
\subsection{Methodology} \label{sec:comparison-methodology}

\subsubsection{Benchmark and validation framework}

Performance and validation comparisons follow a standardized framework:

\begin{enumerate}
  \item \textbf{Reference selection}: For each method, we identify the canonical
    \proglang{R} implementation used in the econometrics literature
    (Table~\ref{tab:reference-packages}).

  \item \textbf{Data generating processes}: All benchmark datasets are generated
    synthetically using identical specifications in both languages, with fixed
    random seeds (seed = 42) for reproducibility. For regression benchmarks:
    \begin{equation}
      y_i = \beta_0 + \beta_1 x_{1i} + \beta_2 x_{2i} + \varepsilon_i, \quad
      \varepsilon_i \sim \mathcal{N}(0, \sigma^2)
    \end{equation}
    Real datasets with published results (Longley, Grunfeld) are used for
    numerical validation.

  \item \textbf{Benchmark execution}: Each method is executed 50--100 times.
    The \pkg{microbenchmark} package \citep{microbenchmark} handles warmup
    and garbage collection in \proglang{R}. \proglang{Rust} benchmarks use
    Criterion \citep{criterion} for automatic warmup and outlier detection.

  \item \textbf{Precision thresholds}: Numerical tolerance varies by sample
    size: $10^{-6}$ for coefficients in small samples ($n < 100$), tightening
    to $10^{-10}$ for large samples ($n > 1{,}000$).
\end{enumerate}

\subsubsection{Robust standard errors}

Heteroskedasticity-consistent standard errors (HC0--HC3) match the
\pkg{sandwich} package \citep{Zeileis:2004} to $< 10^{-8}$ precision
for HC0/HC1 and $< 10^{-8}$ for HC2/HC3 (reflecting differences in
leverage computation).

\subsection{Known differences} \label{sec:known-differences}

A small number of intentional differences exist:

\begin{itemize}
  \item \textbf{Default robust SE}: \pkg{prompt2analytics} defaults to HC1
    (matching Stata), while \pkg{sandwich} defaults to HC3.
  \item \textbf{Panel $R^2$}: We report within $R^2$ by default for FE models.
  \item \textbf{Convergence tolerance}: Iterative methods use $10^{-8}$ default.
\end{itemize}

These differences are documented and do not affect statistical inference validity.
