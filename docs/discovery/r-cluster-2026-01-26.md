## QUICK WINS (High priority + Simple complexity)

These can be implemented quickly with high impact:

### 1. silhouette (Score: 95)
**Description**: Compute silhouette width for cluster validation. Measures how similar a point is to its own cluster vs other clusters.
**Formula**: s(i) = (b(i) - a(i)) / max(a(i), b(i))
**Package**: cluster
**Doc**: https://stat.ethz.ch/R-manual/R-devel/library/cluster/html/silhouette.html
**Deps**: Distance matrix (already have)
**Est. Lines**: ~80

### 2. calinski_harabasz (Score: 91)
**Description**: Calinski-Harabasz Index (Variance Ratio Criterion). Ratio of between-cluster to within-cluster variance.
**Formula**: CH = [BSS/(k-1)] / [WSS/(n-k)]
**Package**: fpc
**Doc**: https://www.rdocumentation.org/packages/fpc/versions/2.2-10/topics/calinhara
**Deps**: Cluster centroids (have from kmeans)
**Est. Lines**: ~50

### 3. davies_bouldin (Score: 90)
**Description**: Davies-Bouldin Index. Average similarity between each cluster and its most similar cluster.
**Package**: clusterCrit
**Doc**: https://www.rdocumentation.org/packages/clusterCrit/versions/1.3.0/topics/intCriteria
**Deps**: Cluster centroids, within-cluster scatter
**Est. Lines**: ~60

### 4. kmedoids/PAM (Score: 83)
**Description**: Partitioning Around Medoids. Like k-means but uses actual data points as centers, robust to outliers.
**Package**: cluster
**Doc**: https://stat.ethz.ch/R-manual/R-devel/library/cluster/html/pam.html
**Deps**: Distance matrix
**Est. Lines**: ~120

### 5. rand_index/ARI (Score: 80)
**Description**: Rand Index and Adjusted Rand Index for comparing clustering solutions.
**Package**: mclust, aricode
**Doc**: https://www.rdocumentation.org/packages/mclust/versions/6.1.1/topics/adjustedRandIndex
**Deps**: None (pure calculation on label vectors)
**Est. Lines**: ~60

### 6. mini_batch_kmeans (Score: 77)
**Description**: Mini-batch K-means for large datasets. Uses random subsets for faster convergence.
**Package**: ClusterR
**Doc**: https://mlampros.github.io/ClusterR/reference/MiniBatchKmeans.html
**Deps**: kmeans (already have)
**Est. Lines**: ~100

### METHOD: Gap Statistic
**Description**: Estimate optimal number of clusters by comparing within-cluster dispersion to null reference distribution.
**Category**: Validation
**Priority Score**: 89/100
**Complexity**: Medium
**Documentation**: https://stat.ethz.ch/R-manual/R-devel/library/cluster/html/clusGap.html

**Existing Related Components**:
- `ml/clustering.rs` - kmeans
- `simulation/` - Random data generation

**Implementation Notes**:
- For each k: compute W_k = sum of within-cluster dispersions
- Generate B reference datasets (uniform or PCA-aligned)
- Compute Gap(k) = E*[log(W_k)] - log(W_k)
- Find k where Gap(k) >= Gap(k+1) - s_{k+1}

**Command to implement**: `/implement_metrics https://stat.ethz.ch/R-manual/R-devel/library/cluster/html/clusGap.html`
